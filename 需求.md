# 六级核心词背诵检测站 — 产品需求（PRD）

> 由 `粗糙想法.md` 整理，并经 `对话 html/index.html` 产品选题更新（2026-05-26）。  
> **技术栈与实现路线** → 见 `plan/技术方案.md`。

## 1. 背景与目标

| 项 | 状态 | 说明 |
| --- | --- | --- |
| 要解决的问题 | [已定] | 考前突击检测每个六级核心词的**中文含义**是否背熟 |
| 成功时用户得到什么 | [已定] | 按批（每批 50 词）刷完词库，看到不熟词回顾与本批/全库用时；全库完成后庆祝、跨次累计总时长；可断点续背、可再开一轮全库 |

## 2. 用户与典型场景

| 项 | 状态 | 说明 |
| --- | --- | --- |
| 目标用户 | [已定] | 备考英语六级的自学者 |
| 典型场景 | [已定] | ① 在**电脑**上本地打开网页，键盘流集中背一批；② 关闭浏览器后下次**接着当前批**继续；③ 分多天背完，庆祝页看**累计总时长**；④ 全库通关后**再背一遍** |

> **说明**：手机端第一版不专门优化（见 §6 `q_mobile_input`）；浏览器能打开即可，不以触屏按钮为必达。

## 3. 功能范围（用户能做什么）

| 项 | 状态 | 说明 |
| --- | --- | --- |
| 核心流程 | [已定] | 加载词库 → 按 CSV 顺序每批 50 词 → 闪卡判词 + 展开中文 → 轮次复习直至本批某一轮 50 词全「记得」→ 本批统计 → **顺序**进入下一批 → 全库庆祝 |
| 全库完成标准 | [已定] | **每一批**都达到「某一轮 50 词全记得」才算全库背完（非仅过完流程） |
| 关键页面 | [已定] | 闪卡页、本批统计页（本轮曾「不记得」词表 + 本批用时）、全库庆祝页（礼花 + 祝贺 + **跨次累计**总时长） |
| 进度与重玩 | [已定] | `localStorage` 记住批次与批内状态；庆祝后可「重新开始全库」 |
| 批次入口 | [已定] | 仅线性 1→2→3…，不可跳批 |

## 4. 体验与交互原则

| 项 | 状态 | 说明 |
| --- | --- | --- |
| 交互方式 | [已定] | 键盘为主：`A`/`←` 不记得、`D`/`→` 记得（同时出中文），已出中文后 `空格` 下一张；未判按空格仅提示；已出中文可改判 |
| 视觉/氛围 | [已定] | 接近 Apple：系统字体栈、大留白、大圆角卡片；英文居中大字，中文下方淡入展开；切词过渡克制 |
| 明暗主题 | [已定] | **仅浅色**（不做跟随系统深色） |
| 手机 | [已定] | 第一版**先不管手机**专设（无大屏触控按钮要求） |
| 进度反馈 | [已定] | 本批位置、本轮剩余「不熟」数；本批/全库用时 |

## 5. 明确不做（范围外）

| 项 | 说明 |
| --- | --- |
| 原生 App、后端、数据库 | 纯前端网页 |
| 艾宾浩斯间隔重复 | 不做跨天智能调度 |
| 英文拼写 / 例句默写 | 只测中文释义 |
| 鼠标点表格选词 | 闪卡 + 键盘流 |
| 批次跳转 / 选批 | 仅线性顺序 |
| 深色模式 | 第一版仅浅色 |
| 手机专用大按钮 | 用户明确第一版不优先 |

## 6. 产品决策记录（HTML 提交）

| 题号 | 问题 | 选择 | 用户可理解含义 |
| --- | --- | --- | --- |
| q_done_definition | 怎样才算「背完」整个词库？ | `every_batch_mastered` | 每一批都要达到「某一轮 50 词全记得」才算全库完成 |
| q_resume_progress | 关闭网页后，下次打开？ | `remember_batch` | 记住进度，下次接着当前批继续背 |
| q_total_time | 庆祝页「总时长」？ | `cumulative_saved` | 跨多次打开累计（分几天背完也合计） |
| q_primary_device | 主要在什么设备上背？ | `desktop` | 以电脑为主，布局与交互优先键盘 |
| q_batch_access | 批次怎么进入？ | `linear_only` | 必须按 1→2→3… 顺序，一批接一批 |
| q_after_complete | 全库背完后还想怎样？ | `celebrate_and_replay` | 庆祝后还能「重新开始全库」再背一遍 |
| q_mobile_input | 在手机上用时需要怎样？ | `__custom__` | **自定义：先不管手机**（不做手机专设交互） |
| q_theme | 界面明暗偏好？ | `light_only` | 只要浅色界面 |

## 7. 成功标准（需求级验收）

1. 每批 50 词内完成轮次复习，且**某一整轮全部判「记得」**后进入统计页，列表含本轮曾「不记得」的词（英+中）与本批用时。
2. 闪卡按键逻辑与改判、空格提示符合 §4。
3. 全库完成 = 所有批次均达到批通关条件；庆祝页有礼花、祝贺文案、**跨会话累计**总时长。
4. 关闭再开后能**续背**当前批；全库完成后可**重开全库**；不能跳批。
5. 仅浅色主题；无后端、无拼写/艾宾浩斯/手机大按钮。
