# M05 — 持久化、全库庆祝与重玩



技术依据：`plan/技术方案.md` §4.3–§4.4；`需求.md` §6（续背、累计时长、再背一遍）。



## 目标



`localStorage` 续背与跨会话累计时长；全库庆祝（礼花、祝贺、总时长）；支持「重新开始全库」。



## 实现要点



1. `js/storage.js`：`load()` / `save()` / `resetLibrary()`。

2. 每次判词、换卡、`beforeunload` 时持久化 `batchSession` 与 `currentBatchIndex`。

3. 累计时长：闪卡学习态计时累加 `totalStudyMs`（见技术方案）。

4. `js/views/celebrate.js`：礼花动画、文案「恭喜完搞定六级所有核心词」、显示 `formatMs(totalStudyMs)`。

5. 「再背一遍」：确认后 `resetLibrary()`，`playthrough++`，**总时长重置为 0**（新一轮计时；历史可选存 `lastPlaythroughMs` 仅开发日志，不对用户展示）。



> **产品说明**：`cumulative_saved` 指单次「从开背到全库完成」路径上的跨天合计；用户点「再背一遍」视为新路径，累计时长清零，避免与上一轮混淆。



6. 打开站点：若有未完成 `batchSession` → 直接进闪卡续背。



## 验收步骤



| # | 操作 | 期望 |

| --- | --- | --- |

| 1 | 背到一半关闭浏览器再打开 | 同一批、队列状态保持 |

| 2 | 隔天继续背完 | 庆祝页总时长包含两天（大于仅本段） |

| 3 | 全库完成 | 礼花 + 祝贺 + 总时长 |

| 4 | 点「再背一遍」 | 从第 1 批开始，总时长从 0 重新计 |

| 5 | 清站点数据 | 等价首次使用 |



**测试键**：任意页面 `Shift + .` 预览庆祝页（不写入完成状态）。
