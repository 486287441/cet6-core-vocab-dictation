# 技术方案 — 六级核心词背诵检测站

> 依据已定 `需求.md`（PRD）。产品决策不在此推翻。

---

## 1. 技术选型结论

| 决策 | 选型 | 理由（对应 PRD） |
| --- | --- | --- |
| 应用形态 | **单页** `index.html` + **原生 ES 模块** `js/*.js` + `css/app.css` | 纯前端、无后端；功能边界清晰，约 8 批 × 50 词规模无需框架 |
| 构建工具 | **无 Vite**（第一版） | 桌面本地 + 静态文件即可；减少配置；后续若加 PWA 可再引入 Vite |
| 词库 | 启动时 `fetch('words.csv')` 解析 | PRD 已定项目内 CSV |
| 持久化 | **`localStorage`** 单键 JSON 状态 | `remember_batch`、`cumulative_saved`、`celebrate_and_replay` 均需跨会话 |
| 部署 | 静态文件；开发用 **本地 HTTP**（`python -m http.server` 或现有 `kami-serve`） | 避免 `file://` 无法 fetch CSV |
| 庆祝动效 | 轻量 **Canvas 粒子** 或纯 CSS（无重型依赖） | 礼花需求；保持单文件可维护 |
| 主题 | 仅浅色 CSS 变量 | `q_theme: light_only` |
| 手机 | 响应式 **可读**即可，不做触控专用 UI | `q_mobile_input` 自定义「先不管手机」 |

## 4. 数据流

### 4.1 词库加载

1. `words.js`：`fetch('words.csv')` → 按行解析 `word,meaning`（处理引号字段）→ `Word[]`。
2. `batchIndex` → `words.slice(i*50, (i+1)*50)`，最后一批可 `<50`。

### 4.2 批内状态（内存 + 持久化快照）

```text
BatchSession {
  batchIndex: number
  round: number                    // 当前轮次，从 1 起
  queue: string[]                  // 当前轮待出现 word id（或索引）
  revealed: boolean                // 当前卡是否已出中文
  currentWordIndex: number
  everForgotten: Map<word, {en, zh}>  // 本批曾判「不记得」
  roundForgottenThisRound: Set       // 本轮暂记不记得（改判可更新）
  batchStartedAt: number
}
```

**轮次逻辑**（与 PRD 一致）：

- 新轮开始：`queue` = 本批全部 50（或末批不足 50）。
- 判「记得」：从本轮 `queue` 移除，本批当前轮不再出现。
- 判「不记得」：记入 `everForgotten`；本轮结束后进入下一轮时，`queue` = 上一轮所有「不记得」词（去重、保持 CSV 顺序）。
- **通关**：某一轮遍历结束时 `queue` 为空且本轮无「不记得」残留 → 等价于本轮 50 词均「记得」→ 进统计页。

（实现时用「本轮结束时 queue 为空」判定，与「整轮 50 词都判记得」等价。）

### 4.3 localStorage（`cet6_vocab_state` v1）

| 字段 | 用途 |
| --- | --- |
| `version` | 模式版本，便于迁移 |
| `currentBatchIndex` | 当前批（续背） |
| `batchSession` | 未通关批的快照（可序列化） |
| `completedBatches` | 已通关批序号列表 |
| `totalStudyMs` | **跨会话累计**学习时长（闪卡活跃时累加，暂停/关页可停止计时策略见 M05） |
| `firstStudyAt` | 首次开始时间戳 |
| `libraryCompletedAt` | 全库完成时间（可空） |
| `playthrough` | 第几轮全库（支持 `celebrate_and_replay` 重置） |

**累计时长**：在闪卡页 `requestAnimationFrame` 或每秒 tick，仅当页面 `visible` 且处于学习态时累加；写入 `totalStudyMs`。

**重开全库**：清空 `completedBatches`、`currentBatchIndex`、`batchSession`，`playthrough++`，保留或重置 `totalStudyMs`（产品建议：**重置累计**并记录历史到 `history[]` 可选；默认 **重置总时长** 以免混淆——在 M05 验收写清）。

### 4.4 路由（hash 或简单 state 机）

| 视图 | 条件 |
| --- | --- |
| `flashcard` | 有未完成批 |
| `batchStats` | 本批刚通关 |
| `celebrate` | 全部批已通关 |
| 重开 | 庆祝页按钮 → 清状态 → `flashcard` batch 0 |

## 5. 与 PRD 对应表

| PRD / §6 | 实现要点 |
| --- | --- |
| 每批 50、CSV 顺序 | `words.js` 切片，批内 `queue` 不乱序 |
| 轮次复习至全记得 | `state.js` 队列 + 轮次递增 |
| 键盘 A/D/空格/改判 | `flashcard.js` + `main.js` keydown |
| 统计：曾不记得 + 本批用时 | `batchStats.js` 读 `everForgotten`、`batchStartedAt` |
| 全库完成 = 每批背熟 | `completedBatches.length === totalBatches` 且每批均通关条件 |
| 礼花 + 祝贺 + 累计总时长 | `celebrate.js` + `storage.totalStudyMs` |
| 续背 | `storage` 恢复 `batchSession` |
| 线性批次 | 仅 `currentBatchIndex++`，无批选择器 |
| 再背一遍 | `storage.resetLibrary()` |
| 仅浅色 | CSS 变量固定浅色 |
| 先不管手机 | 无 `.touch-btn`；`min-width` 媒体查询仅防溢出 |

## 6. 风险与简化

| 项 | 处理 |
| --- | --- |
| CSV 含逗号/引号 | 用简易 RFC4180 解析或仅支持当前 `words.csv` 格式 |
| 改判影响轮次 | 改判只影响当前卡记录与 `everForgotten`，不回溯已结束的轮（与 PRD「已出中文可改判」一致，实现为更新当前词最终判定） |
| 计时精度 | 秒级足够；关页前 `beforeunload` 刷盘 |
