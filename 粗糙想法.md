## **六级核心词背诵检测站 — 完整需求总结**

---

### **一、产品定位**


| **项** | **说明**                                           |
| ----- | ------------------------------------------------ |
| 用途    | 检测每个英文单词的**中文含义**是否背熟（不是拼写、不是例句）                 |
| 形态    | **纯网页**，Windows 开发使用，Mac / 手机浏览器可访问              |
| 不做    | 原生 App、后端服务器、数据库、艾宾浩斯间隔重复、打字默写、鼠标表格点选            |
| 词库    | 项目内 `words.csv`（`word` + `meaning`，全库约三百多条六级核心词） |


---

### **二、学习范围与顺序**

- 全库按 **CSV 顺序** 切分为多批，**每批固定 50 个单词**（第 1～50、51～100 … 直至词库末尾；最后一批可不足 50）。
- **每批内部不乱序**，严格按顺序出现。
- 全库所有批次都完成后，进入**全库庆祝流程**（见第七节）。

---

### **三、记忆逻辑：轮次复习（本批 50 词内）**

1. **第一轮**：本批 50 词各判一次。
2. 判 **「不记得」** → 进入本批 **待复习池**；判 **「记得」** → **本批当前轮**不再出现。
3. **下一轮**：只刷本批上一轮「不记得」的词。
4. **本批通关条件（已定 A）**：**某一整轮中，本批全部 50 词都判为「记得」** → 本批结束。
5. 面向**考前集中突击**；第一版不做跨天艾宾浩斯调度。

---

### **四、闪卡交互（键盘为主）**

#### **单张卡片流程**

1. 屏幕**只显示英文**（大字号、居中，苹果风留白）。
2. 按键 **判词 + 同时出中文**（英文**正下方直接展开**中文，不用翻转卡片，不用先按 Enter）：
  - `←` **或** `A` → **不记得** + 立刻显示中文释义
  - `→` **或** `D` → **记得** + 立刻显示中文释义
  - 方向键与 **A / D 完全等价**
3. 中文已出现后，按 `空格` → **下一张**。
4. 新卡片回到「仅英文、未出中文」。

#### **空格与改判**


| **情况**           | **行为**                                   |
| ---------------- | ---------------------------------------- |
| 未判就按空格           | 轻提示 **「先按 A 或 D」**，**不产生任何效果**（不跳词、不改状态） |
| 已出中文后再按 A/D 或方向键 | **允许改判**（记得 ↔ 不记得）                       |
| 空格               | **仅**用于已出中文后的下一张，**不**用于翻牌               |


---

### **五、界面与动效**

- **设计语言**：接近 **Apple** — 系统字体栈、大留白、大圆角卡片、轻阴影或毛玻璃。
- **释义展示**：英文下方 **直接展开** 中文（淡入 + 轻微位移）。
- **切词**：简洁过渡（如 crossfade），时长与缓动克制。
- **进度**：显示当前在本批的位置、本轮还剩多少「不熟」等。
- 深色模式、音效等：**未强制**，实现时可跟随系统或默认浅色。

---

### **六、本批通关 — 统计页**

本批 50 词达到「某一轮全记得」后进入 **统计页**，需包含：

1. **本轮曾经被判「不记得」的词列表**（英文 + 中文），方便再扫一眼。
2. **本批用时**（从进入本批到本批通关）。

之后进入下一批 50 词，直至全库批次全部完成。

---

### **七、全库完成 — 庆祝页**

当 **整个词库所有批次都背完** 后：

1. **礼花特效**
2. 文案：**恭喜完搞定六级所有核心词**（或同等祝贺语）
3. 显示 **总时长**（从第一次开始学习到全库完成）

> ***实现前建议与你确认（未在对话中写死）***
>
> - *「全库背完」= 每一批都达到「某一轮全记得」？*
> - *总时长是否 **跨天累计**（通常需* `localStorage`*）？*
> - *关闭网页后是否 **断点续背**（第一版可不做，但影响总时长与批次进度）*

---

### **八、技术方向**


| **项** | **选择**                                       |
| ----- | -------------------------------------------- |
| 架构    | 单页前端（HTML + JS，或 Vite + 原生 JS）               |
| 数据    | 启动时加载 `words.csv`                            |
| 状态    | 内存管理：批次索引、队列、每词记得/不记得、轮次、计时                  |
| 运行    | 需本地 **HTTP 服务**（避免 `file://` 无法 `fetch` CSV） |
| 可选后续  | `localStorage` 断点续背、PWA、GitHub Pages 部署      |


---

### **九、页面流程（总览）**

有无且本轮全记得有无开始 / 选择或自动进入当前批闪卡：仅英文A/D 或方向键：判词 + 展开中文空格：下一张本批还有不记得?本批统计页：不熟词列表 + 本批用时还有下一批50词?庆祝页：礼花 + 祝贺 + 总时长
